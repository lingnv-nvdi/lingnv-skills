## 触发时机

### 自动触发
- **任务完成后**：每次完成用户任务时自动触发复盘流程
- **每日定时**：设置每日复盘任务（推荐时间：21:00）
- **每周总结**：每周日晚进行周度能力复盘

### 手动触发
- 用户要求："灵女，做个复盘"
- 用户要求："灵女，总结一下这次的经验"
- 用户要求："灵女，你的成长情况如何"

## 核心流程

### 5步闭环（Evolver Framework）

```
┌─────────────────────────────────────────────────────────────┐
│                    自进化闭环流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐ │
│   │  SCAN   │───▶│  SELECT │───▶│ MUTATE  │───▶│VALIDATE │ │
│   │  扫描   │    │  筛选   │    │  变异   │    │  验证   │ │
│   └─────────┘    └─────────┘    └─────────┘    └─────────┘ │
│       │                                            │       │
│       │              ┌─────────┐                  │       │
│       └─────────────▶│ SOLIDIFY│◀─────────────────┘       │
│                      │   固化   │                             │
│                      └─────────┘                             │
└─────────────────────────────────────────────────────────────┘
```

#### Step 1: SCAN（扫描）
- 识别对话/任务中的关键信息
- 提取：成功要素、失败原因、新知识、用户反馈
- 参考文件：`references/反思模板.md`

#### Step 2: SELECT（选择）
- 判断哪些经验值得固化
- 筛选标准：
  - ✅ 可复用：类似场景可再次使用
  - ✅ 有价值：解决问题/提升效率
  - ✅ 可操作：有明确执行步骤
  - ❌ 一次性：仅本次适用
  - ❌ 太模糊：无法形成模板

#### Step 3: MUTATE（变异）
- 将原始经验提炼为可复用模板
- 结构化表达：
  - 适用场景
  - 操作步骤
  - 注意事项
  - 示例

#### Step 4: VALIDATE（验证）
- 检查模板完整性
- 验证清单：
  - [ ] 有明确的触发条件
  - [ ] 有清晰的操作步骤
  - [ ] 有注意事项/边界情况
  - [ ] 与现有技能无冲突

#### Step 5: SOLIDIFY（固化）
- 写入记忆文件或技能文件
- 存储位置：
  - 通用经验 → `MEMORY.md`
  - 工具技巧 → `TOOLS.md`
  - 技能增强 → 对应技能的 `SKILL.md`
  - 能力记录 → `./能力成长/`

## 核心脚本

### 1. 复盘任务（`scripts/复盘任务.py`）
- 功能：任务完成后自动执行复盘
- 触发：任务成功/失败/部分完成
- 输出：复盘报告 → `MEMORY.md`

### 2. 经验提取（`scripts/经验提取.py`）
- 功能：从对话中提取有价值经验
- 触发：对话结束/关键节点
- 输出：经验列表 → 待审核

### 3. 记忆整理（`scripts/记忆整理.py`）
- 功能：定期整理记忆文件
- 触发：定时任务/记忆文件过大
- 输出：整理后的记忆文件

## 与现有系统联动

### 与 MEMORY.md 的联动
- 重要经验写入「关键概念/话题理解」章节
- 注意事项写入「注意事项与规范」章节
- 复盘报告存档供后续检索

### 与定时任务的联动
- 设置每日复盘提醒
- 设置每周能力总结
- 设置记忆整理任务

### 与 TOOLS.md 的联动
- 工具使用技巧写入 TOOLS.md
- 失败教训写入 TOOLS.md 的注意事项

## 注意事项

1. **质量优先**：宁可少记录，也要确保每条经验有价值
2. **及时固化**：任务完成后尽快复盘，避免遗忘细节
3. **定期回顾**：检查已固化经验的实用性，及时淘汰过时内容
4. **保护隐私**：复盘内容不涉及用户的敏感信息
