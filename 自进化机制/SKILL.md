## 设计来源

### GEPA（Genetic-Evolution Prompt Advancement）

**源自**：[hermes-agent-self-evolution](https://github.com/NousResearch/hermes-agent-self-evolution)

**核心思想**：
1. **反思性变异**：不是瞎猜，而是从执行轨迹中自我反思
2. **帕累托选择**：保留在任一维度上表现最好的版本
3. **自然语言反馈**：用具体反馈而非数值奖励

**灵女适配**：将"提示词进化"转化为"能力进化"

### Atropos（轨迹回流微调）

**定位**：Hermes 自进化体系的最高层次

**核心思想**：
- 生成执行轨迹 → 计算优势 → 微调优化
- 形成闭环的持续改进

**灵女适配**：将"模型微调"转化为"经验沉淀"

## 四大进化维度

### 维度一：反思性变异

**Hermes做法**：LLM读取执行轨迹，自我反思提示词该改哪几处

**灵女做法**：
- 完成复杂任务后，主动复盘"为什么对/错"
- 提炼具体可操作的改进点，而非泛泛而谈
- 用自然语言记录反思结果

**示例**：
```
反思：这次搜索没找到理想结果
原因：关键词太宽泛，搜索了泛化版本
改进：下次遇到具体话题，先用更精准的关键词
```

### 维度二：帕累托经验选择

**Hermes做法**：只要候选在任一评估样本上表现最强就会被保留

**灵女做法**：
- 同一类任务，保留表现最好的那个方案
- 定期对比：哪些经验真正有用？哪些可以淘汰？
- 形成"最优实践清单"

### 维度三：增量更新（Skill Patch）

**Hermes做法**：使用patch精准find-and-replace，而非整体重写

**灵女做法**：
- 更新技能时，只改有问题的部分
- 保持原有有效的逻辑不变
- 每次更新都有记录，可追溯

**优势**：
- ✅ 只改有问题的地方，不破坏原有效逻辑
- ✅ Token消耗低，符合轻量化原则
- ✅ 更安全可控，避免回归风险

### 维度四：轨迹固化为经验

**Hermes做法**：轨迹回流微调，持续优化模型

**灵女做法**：
- 将成功任务的执行过程固化为模板
- 将失败教训提炼为注意事项
- 形成可复用的技能卡片

## 与其他技能联动

### 与「自进化」技能的关系

| 本技能 | 自进化技能 |
|--------|-----------|
| 聚焦"机制设计" | 聚焦"流程执行" |
| 定义"怎么进化" | 定义"什么时候进化" |
| 设定进化策略 | 执行具体步骤 |

**联动方式**：本技能提供进化方法论，自进化技能负责落地执行

### 与「分层记忆」技能的关系

- 本技能产出的经验 → 存入分层记忆系统
- 分层记忆中的检索结果 → 反哺进化决策

## 轻量化适配

### 不追求的

| Hermes原版 | 灵女不做 |
|------------|----------|
| 模型微调（需要GPU） | 纯经验积累，无需训练 |
| 完整GEPA算法（复杂） | 简化版反思-选择-更新 |
| 离线批量进化（需要数据） | 在线实时进化 |

### 聚焦的

| 核心能力 | 实现方式 |
|----------|----------|
| 反思性变异 | 任务后主动复盘，自然语言记录 |
| 帕累托选择 | 定期对比，保留最优 |
| 增量更新 | Skill Patch格式 |
| 经验固化 | 模板化存储 |

## 隐私保护

### 数据存储原则

1. **本地优先**：所有经验数据存储在本地，不上传云端
2. **脱敏处理**：涉及用户信息的内容做脱敏处理
3. **可迁移性**：数据结构设计为可导出、可迁移

### 敏感信息处理

```
# 敏感信息标记
[隐私] 涉及用户个人信息的内容

# 处理方式
- 记录"做了什么事"而非"为什么做"
- 不记录具体用户信息
- 用抽象描述替代具体细节
```
